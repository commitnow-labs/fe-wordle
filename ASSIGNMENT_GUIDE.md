
# 🌱 과제 진행 가이드

> 처음 커밋나우를 이용하시는 분들을 위한 친절한 가이드입니다.
> 아래 순서대로 따라오시면 어렵지 않게 과제를 완료하실 수 있어요!

---

## 1️⃣ 과제 결제하기

마음에 드는 과제를 찾으셨나요? 결제를 완료하면 바로 과제를 시작할 수 있어요.

결제가 완료되면 **두 가지 메일**이 도착합니다:

📬 **커밋나우 결제 완료 메일**

-   구매하신 과제 정보
-   작업 브랜치 이름
-   **제출 마감일** (꼭 확인하세요!)

📬 **GitHub 저장소 초대 메일**

-   GitHub에서 발송되는 초대 메일이에요
-   **반드시 "Accept invitation" 버튼을 눌러 수락**해 주세요
-   수락하지 않으면 저장소에 접근할 수 없어요!

---

## 2️⃣ 저장소 클론하기

GitHub 초대를 수락하셨다면, 이제 로컬 환경에 저장소를 클론해 주세요.

```bash
git clone {저장소_URL}
cd {저장소_이름}
```

> 💡 저장소 URL은 GitHub 저장소 페이지에서 "Code" 버튼을 클릭하면 확인할 수 있어요.

---

## 3️⃣ 작업 브랜치 확인하기

저장소를 클론하면, **작업 브랜치가 이미 만들어져 있어요!**

브랜치 이름은 아래 형식으로 되어 있습니다:

```
work/{세션번호}/{구매일}
```

예를 들어, 구매한 세션의 번호가 1이고 1월 24일에 구매했다면:

```
work/1/2026-01-24
```

> 💡 **세션번호**는 해당 과제의 전체 구매 순서 번호예요. 첫 번째로 구매했다면 1, 두 번째로 구매했다면 2가 됩니다.

해당 브랜치로 이동해 주세요:

```bash
git checkout work/{세션번호}/{구매일}
```

> ⚠️ **중요!** 새로운 브랜치를 만들 필요 없어요. 시스템에서 만들어 둔 작업 브랜치에서만 작업해 주세요.

---

## 4️⃣ 과제 내용 확인하기

과제를 꼼꼼히 읽어보시고 아래 내용을 파악해 주세요:

-   ✅ 과제 내용
-   ✅ 구현 요구 사항
-   ✅ 개발 언어

---

## 5️⃣ 과제 구현하기

이제 본격적으로 코딩을 시작하세요! 🚀

-   커밋은 자유롭게 하셔도 돼요 (횟수, 메시지 형식 제한 없음)
-   중간중간 저장하듯이 커밋하시면 좋아요

```bash
git add .
git commit -m "feat: 로그인 기능 구현"
```

---

## 6️⃣ 작업 내용 푸시하기

구현이 완료되었거나, 중간 저장을 하고 싶다면 푸시해 주세요.

```bash
git push origin work/{세션번호}/{구매일}
```

---

## 7️⃣ Pull Request 생성하기 (= 과제 제출!)

과제 구현이 끝났다면, **Pull Request(PR)를 생성**해 주세요.
PR을 생성하는 순간, **자동으로 과제 제출 + 코드 리뷰 신청**이 완료됩니다!

### PR 생성 방법

1. GitHub 저장소 페이지에서 **"Compare & pull request"** 버튼 클릭
2. 브랜치 설정 확인:

| 항목                      | 브랜치                              |
| ------------------------- | ----------------------------------- |
| **base** (제출 브랜치)    | `submit/{세션번호}/{구매일}` |
| **compare** (작업 브랜치) | `work/{세션번호}/{구매일}`   |

> 💡 `submit` 브랜치도 시스템에서 미리 만들어 뒀어요. 이 브랜치로 PR을 보내야 제출이 정상 처리됩니다!
> 세션번호와 구매일은 작업 브랜치와 동일합니다.

### PR 내용 작성 (선택사항)

PR 제목과 내용은 자유롭게 작성하시면 돼요.

아래 내용을 포함하면 시니어 개발자가 더 좋은 리뷰를 해줄 수 있어요:

-   구현 내용 요약
-   고민했던 부분
-   특별히 피드백 받고 싶은 부분

---

## 8️⃣ 코드 리뷰 받기

PR을 생성하면 끝! 이제 시니어 개발자의 코드 리뷰를 기다리시면 됩니다.

-   시니어 개발자가 **14일 이내**에 코드 리뷰를 진행해요
-   **리뷰가 완료되면 이메일로 알림**이 와요 📬
-   GitHub PR 페이지에서 리뷰 내용을 확인하실 수 있어요

---

## ⏰ 제출 마감 시간

과제마다 제출 기한이 있어요. **마감 시간을 꼭 지켜주세요!**

-   마감: **구매일 + 과제 기한**의 **23:59:59**까지
-   예시: 7일 과제를 1월 24일에 구매 → **1월 31일 23:59:59**까지 제출

> ⚠️ 마감 이후에는 제출이 불가능하고, 코드 리뷰를 받을 수 없어요.

---

## ❓ 자주 묻는 질문

### PR 생성 후에 코드 수정해도 되나요?

네, 가능해요! 하지만 주의할 점이 있어요.

시니어 개발자가 **이미 리뷰를 시작한 후**에 코드를 수정하면,
수정 전 코드 기준으로 리뷰가 진행될 수 있어요.

👉 **PR 생성 전에 최종 점검**하고 제출하시는 걸 추천드려요!

```bash
git checkout work/{세션번호}/{구매일}
```

## 📝 한눈에 보는 과제 진행 순서

```
1. 과제 결제 💳
      ↓
2. 메일 확인 (결제 완료 + GitHub 초대) 📬
      ↓
3. GitHub 초대 수락 ✅
      ↓
4. 저장소 클론 📥
      ↓
5. 작업 브랜치로 이동 🌿
      ↓
6. 과제 내용 확인 📖
      ↓
7. 과제 구현 💻
      ↓
8. 푸시 🚀
      ↓
9. PR 생성 (= 제출 완료!) 🎉
      ↓
10. 코드 리뷰 대기 → 리뷰 완료 메일 수신 📬
```

궁금한 점이 있으시면 언제든 고객센터로 문의해 주세요! 🙌
